## 4. 우선순위 및 구현 순서 권고

### Phase 0 (선행, 작은 PR)
- T2 — `_shared/file-finder.ts:5` `JAVA_EXCLUDE_DIRS`에 `test` 추가 (1줄). 별도 작은 PR.

### Phase 1 (사용자 핵심 페인 — 본 분석의 최대 영향)
- T3 — viewer.html C-1(독립 zoom) + C-2(grid layout). 변경 ~85-130 LOC. 사용자가 첨부 스크린샷에서 직접 본 문제. 가장 가시적 가치.

### Phase 2 (트리 구조 개선 — **Tab2 사전 고지**)
- T4 — `buildNestedSubgraphLines`에서 자식 subgraph를 5개씩 invisible row + `direction LR` 래핑.
- **차단성 위험**: 사용자가 첨부한 두 번째 스크린샷이 정확히 Tab2(BE WINA, 4 rows). 그러나 코드 실증(`mermaid-renderer.ts:313`)에서 Tab2는 route→comp edge를 emit → mermaid "외부 연결 시 direction LR 무시" 제한이 발동될 수 있음. **즉 옵션 A 추천안이 사용자가 본 화면에서는 효과 없을 가능성**이 잔존.
- **분리 진행**:
  - **Phase 2-a**: Tab1 한정 적용 + 실측. 효과 확인되면 commit.
  - **Phase 2-b**: 코드 수정 전 **mini fixture 실증** (route 1개 + comp 1개를 row 래퍼에 넣은 mermaid 코드를 mermaid.live에 붙여보기). 효과 확인되면 Tab2 적용. 효과 없으면 **본 트랙은 Tab1 한정으로 종결** + Tab2는 별도 spike(또는 cytoscape 보류 해제 검토)로 이관.
- 변경 ~15-20 LOC. 활성 spike(`project_mermaid_chunk_ux_spike`)와 직교 — spike 4 후보와 병행 가능.

### Phase 3 (분석 정확도)
- T1 — `route-parser.ts` JsxExpression 분기 + importMap 확장 + `resolveIdentifierToJsxChildren`. 변경 ~118 LOC.
- 단위 테스트 fixture 신규(`fixtures/mini-react-router-jsx-expr`) + 7 케이스.
