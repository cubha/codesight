### A0-3: F-Route-2 (lowercase `component` 키 인식)
- **위치**: `route-parser.ts:67-131` `extractRoutesFromArray`
- **현상**: 인식 키 = `element`(JSX) / `Component`(PascalCase, RR v6.4+ data API) / `lazy`(dynamic import). lowercase `component`는 미인식 → entry 생성되나 elementComponent undefined → renders edge 미생성
- **해소**:
  1. 4번째 분기 추가: `component` PropertyAssignment 발견 시 initializer가 Identifier인지 확인
  2. Identifier name 첫 글자 대문자 가드 적용 (React 컴포넌트 컨벤션, 일반 string/숫자/객체 prop 오인식 차단)
  3. 매핑된 Identifier를 elementComponent로 등록
- **회귀 가드**: 기존 `element`/`Component`/`lazy` 분기 우선순위 유지

### A0-4: F-Route-3 (`<param.prop/>` member access JSX 태그)
- **위치**: `route-parser.ts:229-250` `extractMapPathPrefix` (현재 path만 추출)

- **해소**:

  2. callback parameter name(`route`)과 PropertyAccessExpression의 property name(`component`) 추출
  3. entries 객체에서 동일 키(`component`)의 Identifier 값을 elementComponent로 매핑
  4. F-Route-2 해소 후의 4분기 인식 로직과 결합 (lowercase `component` 키 인식 선행)
- **함수 시그니처 변경 가능성**: `extractMapPathPrefix` 반환에 elementComponent 매핑 정보 추가 또는 별도 helper 분리
